# Vinay AI OS

A personal, single-user AI Operating System: one Next.js app that tracks tasks, career, money, health, learning, coding practice, and a personal knowledge base ‚Äî with Claude generating insight in every module, a per-module Telegram bot for hands-free logging (text or voice), scheduled proactive nudges, and a gamified "Life Score" that rolls all of it into one daily number.

This document describes exactly what the app does today, module by module, field by field, so it can be used as a spec for improvement ideas.

---

## 1. Dashboard (`/dashboard`)

The home screen. Server-computed on every load (`src/features/dashboard/actions.ts`):

- **Life Score (0‚Äì100)** ‚Äî a weighted daily aggregate:
  `health√ó0.25 + finance√ó0.20 + career√ó0.20 + learning√ó0.20 + projects√ó0.15`
  - **Health score**: `(habits done today / total habits) √ó 60` + `(metrics logged today out of 6 fields / 6) √ó 40`
  - **Finance score**: 50 if no budget set; if a budget exists, scored on spend-to-budget ratio (‚â§70% ‚Üí 100, ‚â§90% ‚Üí 85, ‚â§100% ‚Üí 70, ‚â§120% ‚Üí 45, else 20); 60 if zero spend logged
  - **Career score**: profile filled (+25) + `min(25, skills√ó3)` + `min(30, active applications√ó10)` + interview-QA bank non-empty (+20)
  - **Learning score**: `(completed + in-progress√ó0.5) / total resources √ó 100`
  - **Projects score**: `active√ó20 + completed√ó15 + min(25, GitHub push events in last 30 days √ó 2)` (GitHub is optional ‚Äî only counted if `GITHUB_USERNAME`/`GITHUB_TOKEN` are set)
  - Every dashboard load **upserts today's row** into `life_score_logs` (one row per user per day), which is what powers history/trends and the cron digests
- **Gamification**: total XP = sum of every historical daily `life_score`; 9 level thresholds (0 ‚Üí 10,000 XP); a day-streak counter (consecutive days with a logged score); badges (`üå± First Step`, `üìÖ Week Warrior`, `üí™ Month Master`, `üî• 7-Day Streak`, `‚ö° 30-Day Streak`, `‚≠ê Half Century`, `üèÜ Century Club`, `üíé Elite`, `üéØ 1K XP Club`) ‚Äî all recomputed and upserted into `user_xp` on every load
- **30-day score history** chart data (per-module + overall) and a **365-day** window used only for streak/XP
- **Recent bot activity** ‚Äî last 20 rows from `telegram_logs` (what was said to any Telegram bot and how it was actioned)
- Quick stats: pending tasks, active applications, habits done/total today, month spend/budget, learning in progress, active/completed projects, GitHub commit count, document count

## 2. Planner (`/planner`)

Simple task list. Fields: `text`, `done`, `priority` (high/medium/low), `area` (free text, defaults "General"), `due_date`, `recurrence` (daily/weekly/monthly or none).

- Add / delete / toggle-done / change priority
- **Recurring tasks**: completing a task with a `recurrence` set auto-creates the next open instance with the same text/priority/area before marking the current one done
- **Two-way sync with Coding**: toggling a task also updates the linked row in `coding_daily_questions` if the task was auto-generated by the daily coding-challenge system (see ¬ß7), and vice versa
- **AI smart-sort** (`src/features/ai/smart-sort.ts`): sends all tasks to Claude, asks for a re-ordered ID list by urgency/importance/deadline; falls back to original order if the AI response isn't parseable JSON
- **AI focus pick**: asks Claude to name the single most important pending task right now with a one-sentence reason

## 3. Career (`/career`)

Four sub-areas sharing one page:

- **Applications** ‚Äî pipeline: `company`, `role`, `status` (applied ‚Üí screening ‚Üí interview ‚Üí offer/rejected), `salary_range`, `location`, `url`, `notes`, `applied_at`. Add / update status / delete.
- **Career profile** ‚Äî single row: `current_role`, `current_company`, `current_salary`, `target_role`, `years_experience`, `bio`.
- **Skills** ‚Äî `name`, `category` (Frontend/Backend/Testing/DevOps/Architecture/Soft Skills/Other), `level` (beginner‚Üíexpert). Add / change level / delete.
- **Interview Q&A bank** ‚Äî `question`, `answer`, `topic` (JS/TS/React/Angular/Node/Playwright/Testing/System Design/Behavioral/General), `difficulty`. Add / edit answer / delete.
- **AI features**:
  - `career-coach.ts` ‚Äî groups applications by status, gives 4‚Äì5 tactical next steps referencing specific companies by name
  - `career-mentor.ts` ‚Äî a free-form Q&A "mentor" that's fed the full profile + skills-by-category + pipeline snapshot as context (used by an in-app chat, under 250 words, direct verdicts on role-readiness/salary)
  - `generateInterviewQuestions(role, topic, difficulty)` ‚Äî Claude generates 5 realistic Q&A pairs as JSON, presumably to seed the QA bank
  - A second, simpler `getCareerAdvice` also lives in `career-mentor.ts` (status-count summary ‚Üí 3 next steps) ‚Äî duplicate of the one in `career-coach.ts`, worth consolidating

## 4. Finance (`/finance`)

The richest module ‚Äî full personal finance tracking, scoped to the **current calendar month** for expenses/budgets:

- **Expenses** ‚Äî `amount`, `category` (Food/Transport/Housing/Health/Shopping/Entertainment/Learning/Utilities/Other), `description`, `date`. Add / delete.
- **Budgets** ‚Äî one amount per category per month (`upsert` on `user_id,category,month`).
- **Finance profile** ‚Äî `monthly_salary`, `emergency_fund_months`. Changing salary auto-appends a row to `salary_history` (`amount`, `effective_date`) so raises are tracked over time.
- **Loans** ‚Äî `name`, `principal`, `emi`, `interest_rate`, `remaining_months`. Add / delete. Total remaining debt = `Œ£ emi √ó remaining_months`.
- **Investments** ‚Äî `name`, `type` (mutual_fund/stocks/fd/crypto/other), `invested_amount`, `current_value`, `notes`. Add / update current value / delete. P&L = current ‚àí invested.
- **Financial goals** ‚Äî `name`, `target_amount`, `current_amount`, `target_date`, `priority`. Add / update progress / delete.
- **Rolling 3-month average expense** ‚Äî computed on every load from the last 90 days of expenses, used as the "realistic monthly spend" baseline (as opposed to just this month's partial total).
- **AI advisor** (`finance-advisor.ts`) ‚Äî free-form Q&A fed the full financial snapshot: salary, EMIs (itemized per loan), 3-month avg spend, free cash/month (`salary ‚àí EMIs ‚àí avg spend`), portfolio with per-holding P&L, goals with target dates, and emergency-fund target in ‚Çπ. Answers under 200 words, numbers-driven, gives clear verdicts on affordability/investment questions.

## 5. Health (`/health`)

Two systems layered together: **habits** (streaks) and **metrics + weight-loss coaching**.

- **Habits** ‚Äî `name`, `icon`(emoji). Log/unlog per date; a 7-day rolling log window is fetched for the streak grid.
- **Daily metrics** (`health_metrics`, one upsertable row per user per day) ‚Äî `weight_kg`, `calories`, `protein_g`, `sleep_hours`, `steps`, `water_ml`, `notes`.
- **Health profile** ‚Äî `age`, `gender`, `height_cm`, `target_weight_kg`, `activity_level` (sedentary‚Üívery_active), `workout_days_per_week`, `food_preference`, `goal_deadline`.
- **Calculations** (`calculations.ts`, pure functions, no AI):
  - BMI, Mifflin-St Jeor **BMR**, **TDEE** (BMR √ó activity multiplier 1.2‚Äì1.9)
  - **Weight-loss plan**: given current/target weight and TDEE, computes weekly loss rate (capped at 1kg/week, respects a goal deadline if set or defaults to a safe pace), daily calorie target (floored at 1500 kcal), protein target (2g/kg bodyweight), and a 25%-fat / remainder-carbs macro split ‚Äî plus an expected goal-completion date
  - **Health Score breakdown** (0‚Äì100, distinct from the Dashboard's health sub-score): `nutrition√ó0.3 + activity√ó0.25 + sleep√ó0.25 + consistency√ó0.20`, each sub-score carrying a plain-English reason string (e.g. "Protein is 24g below target", "7h is below the 7‚Äì9h target"). Activity score also detects habit names matching `/gym|workout|run|walk/i` as a proxy for "worked out today."
- **AI features**:
  - `getHealthReport` ‚Äî weekly trend report (weight change over the period, sleep/calorie/protein/step/water averages) with a /10 score, what's working, what isn't, 3 recommendations
  - `getDailyHealthPlan` ‚Äî today's action checklist: remaining calories/protein/water to hit target, habit status, current Health Score with reasons, output as an emoji checklist under 150 words, plain text

## 6. Learning (`/learning`)

Resource tracker: `title`, `type` (course/book/video/article/podcast), `url`, `category`, `status` (not-started/in-progress/completed), `progress` (0‚Äì100), `notes`.

- Add / update status+progress+notes / delete
- **Study logs** ‚Äî `resource_id`, `duration_minutes`, `notes`, `date`; last 30 days fetched with the page
- **AI daily study plan** ‚Äî given in-progress/not-started/completed counts and what's already been studied today, Claude proposes a "main focus" (60 min, references an actual resource by name), a "quick review" (15 min), and an optional stretch item ‚Äî under 150 words
- **AI resource quiz** ‚Äî `generateResourceQuiz(title, category, type, notes)` generates 5 comprehension questions (mixed conceptual/applied/comparison) as JSON, presumably for self-testing after finishing a resource

## 7. Coding (`/coding`)

Two halves: a **project tracker** and an automated **daily coding-challenge habit system**.

- **Projects** ‚Äî `name`, `description`, `status` (idea/in-progress/paused/completed), `stack` (tag array), `github_url`, `live_url`. Add / update status / delete.
- **Daily coding challenge** (`daily-core.ts`, driving `coding_daily_questions` + `coding_questions` tables):
  - Two modes, per-user in `coding_settings`: **rotation** (weekday-based difficulty mix ‚Äî Mon: 1 easy, Tue: easy+medium, Wed: 2 medium, Thu: medium+hard, Fri: 2 medium, Sat: 1 hard, **Sun: none ‚Äî revision day**) or **fixed** (always N random questions/day, no difficulty pattern)
  - Picks random unassigned questions from the `coding_questions` pool per difficulty bucket; if a difficulty bucket is exhausted, the cycle restarts (questions can repeat once the pool is exhausted)
  - Idempotent per day: calling it again the same day returns the existing assignment instead of generating a new one
  - **Each assigned question auto-creates a linked Planner task** (`area: "Coding"`, priority "high" for hard questions else "medium") ‚Äî this is the mechanism behind the Planner two-way sync in ¬ß2
  - **Stats**: current streak, longest streak (both computed by walking `assigned_date`/`completed` history day-by-day; today is allowed to be pending without breaking the current streak), total/easy/medium/hard solved counts, completion rate
  - **Calendar view** (`computeCodingCalendar`) ‚Äî up to 182 days back, each day classified `solved` / `partial` / `missed` / `none` for a GitHub-style contribution heatmap
  - `telegram_notify` setting can turn off the daily Telegram push (¬ß10) without disabling the assignment logic itself

## 8. Documents (`/documents`)

A personal full-text knowledge base: `title`, `content`, `tags` (array), `summary`, `updated_at`.

- Add / edit / delete
- **Auto-summarization**: every create/update that has ‚â•300 characters of content triggers an async Claude call (`generateSummary`, capped at first 6000 chars) to write a 2‚Äì3 sentence summary stored alongside the document ‚Äî silently skipped (empty string) if content is short or the AI call fails
- **AI document Q&A** (`doc-qa.ts`): ask a question about one specific document; Claude answers strictly from that document's content (truncated to 6000 chars) and says explicitly if the answer isn't in the text
- **AI document summarization** ‚Äî on-demand 3‚Äì5 bullet-point summary of a document (distinct code path from the auto-summary above, used interactively)

## 9. Search (`/search`)

Cross-module search input (`SearchInput.tsx`) ‚Äî a single query surface over the modules above.

## 10. Telegram bot ‚Äî the interaction layer

This is the most distinctive part of the system: **each module is its own Telegram bot** (own `TELEGRAM_BOT_TOKEN_*`), all pointed at one webhook handler (`src/app/api/telegram`) that dispatches by module name to `src/features/telegram/handler.ts`.

How a message is processed:
1. Only messages from `TELEGRAM_ALLOWED_CHAT_ID` are accepted ‚Äî this is a single-user, single-chat system, not multi-tenant
2. **Voice messages** are transcribed first via Groq (`transcribeVoice`, needs `GROQ_API_KEY`) before anything else happens ‚Äî so every module is fully usable by voice note, not just text
3. The transcribed/typed text is sent to Claude along with a **module-specific system prompt** that defines a small JSON action grammar (e.g. `{"action":"add_expense","amount":500,"category":"Food",...}`) plus the *actual current date* injected into the prompt (so "today"/relative dates never rely on the model's training cutoff)
4. Claude's JSON response is parsed and dispatched to that module's `execute()` function, which does the real Supabase read/write and formats a Markdown reply (with emoji category/status indicators)
5. Every interaction ‚Äî module, chat id, raw message, parsed action, and reply ‚Äî is logged to `telegram_logs` (used by the Dashboard's "recent bot activity" feed); logging failures are non-fatal

**Per-module natural-language capabilities:**

| Bot | Example phrases it understands | Actions |
|---|---|---|
| **Planner** | "add buy groceries high priority", "show pending tasks", "done with buy groceries", "delete buy groceries", "add call dentist due 2026-07-10" | add/list/complete/delete task |
| **Career** | "applied to Google as Frontend Engineer", "Google moved me to interview", "show all applications", "add note to Google: good culture fit", "pipeline summary" | add/update-status/list application, add note, status summary |
| **Finance** | "spent 500 on Swiggy food", "show today's expenses", "monthly summary", "set food budget 8000", "net worth", "my salary is 120000", "add home loan 20L EMI 15000 180 months", "add SIP Axis Bluechip invested 50000 current 65000", "can I afford a car?", "should I prepay my loan?" | expense add/list, monthly summary, set budget/salary, add loan/investment, net-worth snapshot, free-form AI advisor Q&A |
| **Health** | "weight 88kg", "slept 7.5 hours", "8000 steps", "2000 calories", "120g protein", "2L water" (auto-converted to ml), "logged my run" (fuzzy-matches an existing habit), "did meditation", "show habits", "today's summary", "add habit Journaling üìî" | log metric, log/list habit, add habit, today's metrics/summary |
| **Learning** | "add Next.js course from Udemy", "started JavaScript: The Good Parts book", "update Next.js to 60%", "finished React docs", "show in-progress resources" | add resource, update progress, mark complete, list by status |
| **Coding** | "add project Portfolio with Next.js and Tailwind", "started working on Portfolio", "show in-progress projects", "Portfolio is done", "add note to Portfolio: deploy to Vercel next", "today's question", "solved Two Sum" | project CRUD, fetch/complete today's coding challenge (fuzzy title match) |
| **Documents** | "note that Next.js 15 uses server components by default", "create doc Interview Prep with content...", "add to Interview Prep: practice system design", "search React hooks", "list documents" | create/append/search/list documents |

Any message the model can't confidently map to an action falls back to `{"action":"help"}`, which each bot answers with its own cheat-sheet of example phrases (shown in the table above).

## 11. Scheduled jobs (Vercel Cron)

Defined in `vercel.json`, all protected by a shared-secret `Authorization: Bearer $CRON_SECRET` header, all resolving the single app user via `supabase.auth.admin.listUsers()[0]` (this is a single-user deployment, not multi-tenant):

| Job | Schedule (UTC) | What it does |
|---|---|---|
| `daily-briefing` | `0 3 * * *` (~8:30am IST) | Recomputes today's Life Score components, compares to yesterday's, and has Claude write a <120-word motivating morning message (score trend, single most important action, one thing to watch) ‚Äî sent via the Planner bot |
| `daily-coding` | `5 3 * * *` (~8:35am IST) | Generates (or fetches) today's coding-challenge assignment and pushes it via the Coding bot, respecting the per-user `telegram_notify` setting; silently no-ops on revision days or an empty pool |
| `evening-checkin` | `30 14 * * *` **and** `0 17 * * *` (8:00pm and 10:30pm IST) | Checks for anything still un-logged today ‚Äî pending habits, unfinished high-priority tasks, no expense logged, missing health metrics (weight/sleep/steps/calories) ‚Äî and only sends a message if something is actually outstanding; stays silent once everything's logged |
| `weekly-digest` | `30 2 * * 0` (Sunday ~8am IST) | Pulls the last 7 days from `life_score_logs`, computes averages per module, identifies the strongest/weakest module and best/worst day, renders an ASCII progress-bar scorecard, and has Claude write a 3-sentence weekly review |

All four use the Planner bot's token to send (`TELEGRAM_BOT_TOKEN_PLANNER`), regardless of which module they're reporting on.

## AI model & failure handling

Every AI call funnels through one helper, `src/lib/anthropic.ts` ‚Üí `aiText(prompt, system?)`, using `claude-sonnet-4-6` with a 4096-token cap. Almost every call site wraps this in `.catch(() => '')` (or an equivalent empty-array fallback for JSON-returning calls), so **a missing/invalid `ANTHROPIC_API_KEY` degrades every AI feature to silently absent output rather than crashing a page or a cron job.**

## Stack

- **Framework**: Next.js 15 App Router (TypeScript)
- **Auth + DB**: Supabase (PostgreSQL + Row Level Security)
- **Styling**: Tailwind CSS v3 with a custom `surface` / `accent` color system
- **Components**: shadcn/ui (Radix UI primitives)
- **AI**: Anthropic Claude (`claude-sonnet-4-6`) via `@anthropic-ai/sdk`
- **Voice transcription**: Groq (for Telegram voice notes)
- **Bot**: Telegram Bot API webhooks, one bot token per module
- **Deploy**: Vercel (auto-deploy from `master`), Vercel Cron for scheduled jobs

## Architecture

**Thin page + feature view pattern:**
- `src/app/[route]/page.tsx` ‚Äî async server component, fetches data, passes to view
- `src/features/[module]/components/[Module]View.tsx` ‚Äî `'use client'` component, owns all interactivity
- `src/features/[module]/actions.ts` ‚Äî `'use server'` functions (CRUD via Supabase)
- `src/features/[module]/types.ts` ‚Äî TypeScript types for the module

**Optimistic UI:** mutations use `useOptimistic` + `useTransition` so the UI updates instantly before the server confirms.

**Supabase clients:**
- `src/lib/supabase/server.ts` ‚Äî server components and server actions (cookies-based, RLS-scoped to the logged-in user)
- `src/lib/supabase/client.ts` ‚Äî client components (browser)
- `src/lib/supabase/service.ts` ‚Äî service-role client (used by cron jobs and the Telegram webhook ‚Äî bypasses RLS since those code paths run without a browser session)
- `src/lib/supabase/middleware.ts` ‚Äî session refresh + redirect logic, wired in root `middleware.ts` (matcher excludes `/api`, `_next/static`, `_next/image`, favicon, and image assets ‚Äî so Telegram/cron webhooks are never gated by the browser auth check)

**Loading / error states:**
- `src/app/[route]/loading.tsx` ‚Äî skeleton shown while the server fetches data
- `src/app/[route]/error.tsx` ‚Äî error boundary with a "Try again" reset button

**Shared components** (`src/components/`): `Sidebar`, `Header`, `Card`, `UserInfo`, `Skeleton`.
